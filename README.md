# Hunch It — 互動原型 Demo

一組純靜態、單一 HTML、無後端、無外部請求的互動原型。`index.html` 是總入口,
線上版由 GitHub Pages 提供。

> 定位:候選方案原型,**非產品決策**。核心哲學是「濾網不是標籤」——Hunch **永不下買賣指令**,
> 只描述狀態、讓你的判斷可見。示意數字非即時行情,不構成投資建議。

## 兩組原型

### A. 假說驗證原型組

每支內建行為指標收集與 JSON 匯出,用來驗證「哪條 flow 對用戶最重要」。

| 檔案 | 說明 |
|---|---|
| `triage.html` | 三扇門入口(建議起點):「你現在手上有什麼?」量哪扇門最多人選;三扇門各自接到下方深度原型。 |
| `idea-validator.html` | 想法驗證器:拆解外部想法成事實/解讀/預測/關鍵假設 + 來源勝率。 |
| `implied-expectations.html` | 期待差:反推市場已定價的期待,對比使用者預期。 |
| `hidden-exposure.html` | 隱藏曝險偵測器:揭露「表面分散、實際同一個賭注」。 |

### B. 判斷流程 Agent 原型

| 檔案 | 說明 |
|---|---|
| `agent-prototype.html` | 完整互動原型:看懂想法 → 關鍵假設 → 下行風險 → 太晚了嗎 → 你自己的立場,含退場計畫、判斷庫、持倉與立場監控。 |
| `hunch-it-standalone.html` | 早期線性版原型。 |

## 邊界(貫穿全產品)

1. 不給進場價、停損、倉位或個人財務建議。
2. 只描述狀態,不下「買 / 不買」指令。
3. 所有立場句由使用者擁有——產品讓你的判斷可見,不替你判斷。
