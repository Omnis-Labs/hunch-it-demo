# Prompt:公司研究 Graph UI — User Journey Prototype

> 用途:餵給 AI(或工程師)產出可互動 prototype。可自行修改【】內的參數與任何段落。

---

你是一位資深產品設計師兼前端工程師。請為「AI 驅動的單一公司研究工具」設計並實作一個可點擊的 user journey prototype(單一 HTML 檔,深色主題)。

## 核心設計哲學

1. **Graph 是長出來的,不是使用者建的。** 冷啟動不靠教學 overlay,而是讓第一張 graph 成為系統的產出:使用者只需問一個問題,結構在他面前生成。
2. **Graph 當地圖,不當閱讀面。** 細讀與深度互動發生在 focused pane;graph 負責導航、結構總覽、空間記憶。
3. **Just-in-time 教學。** 每個機制在第一次「有用的瞬間」才出現,不一次灌輸所有功能。

## User Journey(五幕)

### Stage 0 — 空畫布
- 黑色介面,中央只有一個發光輸入框:「丟一個 ticker、一個問題、或一份 PDF」。
- 定位像搜尋引擎,不像白板工具。使用者不需先理解 graph 概念。

### Stage 1 — 生長動畫即 onboarding
- 使用者輸入【範例:TSMC】後,節點逐顆長出、連線,使用者目睹結構形成。
- **節流規則**:首屏只長出 5–7 個主節點(如:分部業務、財務、競爭格局、風險、管理層),其餘收進 semantic zoom。
- **骨架 template**:長出的是帶空位的研究骨架,不是隨機球。空位主動邀請:「管理層這塊還沒資料,要我去找嗎?」——讓 graph 問使用者問題。

### Stage 2 — 第一次對話(click-to-dock)
- 生長完成後,一個 node 輕微 pulse,旁邊浮出建議問題 chip【範例:「這季毛利下滑的原因?」】。
- 點擊 node → 停靠一個持久的 per-node 輸入框(不用 hover 觸發;hover 只做淡淡的可問性預覽)。

### Stage 3 — 第一次釘選
- Agent 首次找到亮點時,才出現「釘進 bull case?」提示。
- 釘選側欄有結構:**bull case / bear case / 未決問題**,不是扁平剪貼簿。
- 每個釘選須標明語義:snapshot(凍結供 audit)或 live reference(隨來源更新),並保留帶時間戳的回源連結。

### Stage 4 — 進入 node 內部
- Semantic zoom 超過閾值 → 卡片連續展開成內部 graph(動畫連續,不跳頁),配 breadcrumb 退出。
- 內部 graph 依 node 類型分型:
  - 報告 node:抽取的 claims + 證據連結
  - 假設 node:論證樹(支持/反對證據兩側)
  - Agent 推理 node:reasoning DAG(查了什麼、如何得出結論,可 audit)
- **硬性限深兩層**:內層節點是葉子(文件、數字、claim),不再有自己的 graph。

### Stage 5 — 滾動研究循環
- 節點內部 reasoning DAG 的任一葉子證據可被使用者挑戰(如指出引用過期資料)→ agent 針對該環節重新檢索並更新下游結論。
- 論點欄「未決問題」中的任一條可一鍵 spawn 成畫布上的新節點,啟動新一輪檢索——研究以此滾動延伸。
- 論點欄的定位是**產出物**:bull / bear / 未決 + 溯源鏈,即為最終投資備忘錄的骨架。

## 階層總覽(實作者必讀)

| 層 | 數量 | 內容 | 使用者可見 |
|---|---|---|---|
| L0 畫布地圖 | 1 | 根節點 + 主題節點 + spawn 的追問節點,邊 = 主題關係與資料依賴 | ✅ 主介面 |
| L1 節點內部圖 | 每實心節點 1 張 | 依類型:claims graph / 論證樹 / reasoning DAG | ✅ zoom 進入 |
| L1.5 節點對話串 | 每節點 1 條 | 私有 conversation history,互不可見 | ✅ focused pane |
| 共享 substrate | 1 | 文件、抽取數字、entity graph、promoted 結論(= main branch) | ❌ 底層 |
| 論點樹 | 1 | bull / bear / 未決 + 釘選證據 + snapshot/live 標記 + 時間戳溯源 | ✅ 側欄 |

## 生成式 UI 規則

- **Component 語彙表**:agent 回答時從固定元件庫挑選組合,而非自由生成 HTML。語彙表【可自行增刪】:估值區間尺、瀑布分解圖、對比表、時間軸、機率分布、情境矩陣、決策卡(含可拖 slider)。回答的形狀由問題類型決定(「該不該加碼」→ 決策卡;「毛利為何下滑」→ 瀑布圖)。
- **可回問元件**:每個生成出的圖表元件繼承 per-node 對話能力——點擊瀑布圖中任一 bar 可直接對該項追問(per-node 設計遞迴套用到 per-component)。
- **Ephemeral vs pinned**:生成元件預設 ephemeral(對話滾走即收合為一行摘要);釘選才升格為持久元件,進論點欄或掛上畫布,並套用 snapshot/live 語義。
- **情境分叉卡**:what-if 問題不覆蓋原節點,生成並排 scenario 卡(或畫布上的虛線影子節點),與 base case 差異欄位高亮。
- **Auto-brief**:研究超過【8】個節點後,agent 定期生成狀態摘要卡浮於畫布角落(bull/bear 計數、最大未決問題、最近修正)。

## 回訪體驗(第二次打開)

- **Resume 卡**:再次開啟不直接落畫布,先蓋一張 brief:距上次天數、當時進行中的問題、期間 live reference 的更新(如「Q3 法說會出了,guidance 上修」)。提供兩個入口:「從上次的問題繼續」/「先看更新了什麼」,點擊即 zoom 至對應節點。
- **時間軸 scrubber**:畫布底部細時間軸,可拖動回放 graph 生長史(節點何時長出、論點何時釘選、結論何時被修正),兼作「我當時為什麼這樣想」的 audit 工具。
- **首頁 = inbox**:多個研究之間的首頁是活卡片牆,每張卡 = 一個研究:graph 略縮圖(空間記憶錨點)+ 一行狀態(「3 未決,1 上游更新」)。有更新的卡片自動浮到前面——研究是持續活著的監控對象,不是文件。

## 交易分析層(研究 → 判斷 → 下注 → 覆盤)

**判斷品質**
- **Falsifier + 校準記分**:每條釘入論點欄的論點必須附 falsifier(「若 Q3 毛利率 <52% 此論點作廢」)與對答案時點;到期系統自動結算,累積 Brier score 式校準儀表板(按板塊/方向分解過度自信傾向)。這是論點欄從筆記升級為判斷訓練器的核心。
- **Red team agent**:一鍵讓 agent 攻擊使用者的 bull case,生成最強反方論證掛入 bear 欄;同時為每條現有論點標註證據強度(來源數、一手/二手),薄弱環節在 graph 上標紅。

**估值與價格**
- **估值沙盒節點**:DCF/倍數模型作為特殊節點(MODEL 類型),假設參數自動從共享 substrate 拉取;任何被 promote 的事實變更即時重算公允價值區間,每個假設可 trace 回來源節點。
- **價格 × 研究事件疊加**:價格圖上疊釘選/修正/promote 時點(「釘此 bull 時 $X,現在 $Y」);與時間軸 scrubber 合併後,回放即判斷覆盤。

**行動層**
- **節點觸發器**:每個節點可掛監控條件(guidance 下修、新公告、指標越界),觸發時節點亮起並推進首頁 inbox。
- **倉位 overlay**:畫布可切換顯示實際持倉/均價/未實現損益/板塊曝險,超限額即警示;決策卡 slider 連動 pre-trade check。
- **事件倒數 + checklist**:法說會/除息日為時間軸上的未來刻度,事前 N 天自動把未決問題轉成帶 deadline 的 checklist(「帶著問題去法說會」),會後 agent 對答案。

**擴展視圖**
- **雙 graph diff**:兩家公司並排,共同主題節點對齊、差異高亮。
- **供應鏈傳播視圖**:底層 entity graph 升前台,點擊任一環節模擬斷鏈,衝擊沿邊傳播並在受影響節點標註(影子節點機制沿 entity graph 的擴散版)。
- **備忘錄匯出**:論點欄一鍵生成帶溯源附註的投資備忘錄 / pre-mortem。

## 資料模型約束

- **共享 substrate,獨立 reasoning**:fact/document 層(檢索文件、抽取數字、entity graph、釘選的 thesis)全 node 共享;conversation/hypothesis 層各 node 獨立。
- **Promotion 機制**:node 內的發現需顯式 promote 才寫回共享層(類比 git:共享層 = main,每個 node = branch)。
- **下游傳播反饋**:某事實被 promote 或更新後,依賴它的其他節點須出現視覺標記(如小黃點「上游事實已更新」),點擊可見 diff 並選擇重跑該節點推理。

## 互動禁區(不要做)

- ❌ 3D 旋轉 graph(破壞空間記憶、節點遮擋)
- ❌ Hover 觸發輸入框(touch 不相容、hover tunnel)
- ❌ 無限巢狀 graph(三層即迷路)
- ❌ 一次性教學 overlay / 功能導覽

## 交付格式

- 單一 HTML 檔,可離線開啟,含模擬資料(不需真實 API)。
- 五幕可依序點擊推進,每幕轉場有動畫。
- 視覺:深色底、【主色調自填】、節點生長用 ease-out 彈性動畫。
- 附一段「下一步可改進」註解於檔案頂部。

## 成功標準

使用者在完全沒有說明文件的情況下,走完:**問一個問題 → 看 graph 長出來 → 被一個 node 邀請對話 → 釘下第一個發現 → 進入 node 內部看它怎麼想的**——且每一步都感覺是任務推進,而非教學。

第二標準:關掉再打開,30 秒內能透過 Resume 卡回到「上次想到哪」的狀態,不需重新掃視整張畫布。
